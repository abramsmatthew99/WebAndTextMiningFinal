<strong>Description<br/><br/></strong>This role will support the identification, creation &amp; reporting of data metrics used to enforce Information Security policies and programs. These metrics are reviewed and tracked by Process Owners to drive security compliance &amp; governance, control implementations, and allow process improvements.<br/><br/>Ideally targeting someone experienced in data warehousing and Power BI. This person will need to be skilled in identifying &amp; analyzing process data, sourcing &amp; extracting data from disparate sources, and creating dashboards.<br/><br/>This person will need to build datasets necessary to generate policy-tracking metrics and executive-level dashboards of these metrics.<br/><br/><strong>Responsibilities<br/><br/></strong>Our client has a backlog of work and needs a Data Analyst to:<br/><br/>Identify policy and security program metrics to demonstrate compliance of security protocols<br/><br/>Further mature current policy metrics to include the new aspects of policies, enhancing process tracking by using more data<br/><br/>Develop data necessary to track 20 separate policies /standards for governance and compliance, all coming from many unique data sources.<br/><br/>Create executive-level dashboards for each of the non-compliant policy models, combining many disparate data sources into one Power BI summary<br/><br/>The data sources are unique to each policy and many times the data needed is housed in ‘systems’ such as InTune, Audit log reports, Active Directory, Service Now, One Trust, CourseMill, SharePoint site activity, Social Chorus, Knowbe4, and Cofense.<br/><br/><strong>In order to measure compliance, for each policy it is necessary to<br/><br/></strong>identify the source system(s) housing the data that will give us the policy compliance insights – which sometimes requires process changes to capture the data, and<br/><br/>Pulling the data into Power BI for modeling by gaining direct access to the system, extracting data from the system, and building a historical dataset (this is the case the majority of the time).<br/><br/><strong>Requirements<br/><br/></strong>Ability to source &amp; extract data from disparate sources for a specific purpose<br/><br/>Experience with Informatica to create ETL pipelines<br/><br/>Data cleansing and preparation<br/><br/>Data analysis and exploration<br/><br/>Create data visualizations<br/><br/>Create dashboards, analytics, and reports<br/><br/>Critical thinking<br/><br/>Problem solving<br/><br/>Attention to detail<br/><br/>Business strategy and operations understanding<br/><br/>Writing and communication<br/><br/>Statistical knowledge<br/><br/>Domain/Active Directory knowledge<br/><br/>Programming language (specifically SQL, Python, and R)<br/><br/>SSIS is a plus<br/><br/>knowledge of using and working with APIs<br/><br/>Cyber Security metrics experience is a plus<br/><br/>Masters of Science in Business Informatics a plus<br/><br/><strong>Education:</strong> Bachelors Degree
